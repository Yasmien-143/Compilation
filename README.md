This subject consists of the following activities/projects:
**Activity 2 – UML Diagram Design Project (Co-Fit: Gym Operations and Membership System)**  

This activity involves designing a gym management system called “Co-Fit: Gym Operations and Membership System” using UML concepts. The project focuses on creating a system that helps manage memberships, workout schedules, attendance tracking, billing, and administrative operations for a fitness center. The system identifies three main actors: Customer, Trainer, and Admin, each with specific roles and functions within the platform. A Use Case Diagram was created to visually represent the interactions between users and the system, helping improve understanding of system processes, functionality, and overall software design.





**UML Final Project**

This activity focuses on designing and analyzing the “Co-Fit: Gym Operations and Membership System” using UML concepts and system development principles. The project aims to solve common gym management problems such as manual record keeping, scheduling conflicts, payment tracking issues, and inefficient member monitoring through a centralized digital platform. The system includes features for membership management, workout scheduling, attendance tracking, payment processing, and report generation for members, trainers, and administrators. A UML Use Case Diagram was created to illustrate the interaction between users and the system, helping improve understanding of system processes, functionality, and software design structure.
